El proyecto se llama Finaxis y es una aplicación web de análisis financiero orientada a instituciones bancarias y financieras, pero también con un módulo educativo para docentes y alumnos. El stack es React con TypeScript usando Vite como bundler, Tailwind CSS para estilos, shadcn/ui para componentes, y Supabase como backend completo con autenticación, base de datos PostgreSQL y storage.
El sistema permite que cualquier persona se registre con una sola cuenta y maneje múltiples perfiles, similar a Netflix. Los perfiles disponibles son Usuario Personal, Docente y Alumno. El usuario puede cambiar entre perfiles desde una pantalla de selección al ingresar.
El perfil de Usuario Personal puede subir su propio catálogo de cuentas en formato de archivo, crear fórmulas personalizadas de ratios financieros, definir sus propios límites de riesgo por niveles (bajo, moderado, por encima del promedio y alto), hacer análisis vertical y horizontal, y exportar todo en PDF.
El perfil Docente puede crear ejercicios o casos prácticos con estados financieros reales o ficticios, subir el catálogo de cuentas para cada caso, asignar ejercicios a sus alumnos mediante código de invitación o email, ver en tiempo real lo que hace el alumno incluyendo qué fórmulas usa y qué ratios calcula, configurar el modo examen con límite de tiempo y bloqueo de fórmulas guardadas, y evaluar los resultados finales.
El perfil Alumno puede analizar los estados financieros que el docente le asigna, guardar sus propias fórmulas, pero en modo examen esas fórmulas quedan bloqueadas e invisibles. Al terminar el examen puede exportar su análisis en PDF para entregarlo.
El catálogo de cuentas sigue la estructura jerárquica estándar de la Superintendencia de Bancos, por ejemplo 1 es Activos, 11 es Fondos Disponibles, 1101 es Caja, 110105 es Efectivo. Muchas cuentas pueden estar en cero pero igual se carga todo el catálogo completo.
Los análisis que soporta la app son análisis vertical donde cada cuenta se divide entre el total del grupo para obtener su porcentaje de participación, análisis horizontal donde se comparan dos períodos y se calcula la variación absoluta y porcentual, y ratios financieros como razón corriente, prueba ácida, endeudamiento sobre patrimonio, entre otros que el usuario puede crear y guardar libremente.
Para los ratios el usuario escoge qué cuentas del catálogo usa en cada parte de la fórmula, la app no impone una fórmula fija, el usuario la construye. Cada ratio tiene niveles de riesgo configurables por el propio usuario porque cada institución maneja rangos distintos.

Nuevas implementaciones técnicas y de autenticación (Finaxis):
- Flujo completo de registro (con validación de OTP) e inicio de sesión usando Supabase Auth.
- Validación manual de correos duplicados consultando directamente la tabla `personas` antes del registro para manejar el comportamiento por defecto de Supabase de no revelar correos existentes.
- Flujo de Google OAuth optimizado usando una ventana emergente (Popup) y la API `BroadcastChannel`. Esto previene el bloqueo por la política COOP (Cross-Origin-Opener-Policy) y evita ensuciar el historial de navegación del usuario con redirecciones externas.
- Control estricto del historial del navegador para los flujos de autenticación: se utiliza `window.location.replace()` a nivel de sistema operativo para las redirecciones post-login, registro y cierre de sesión. Esto purga la pila del historial, impidiendo que el usuario pueda retroceder a pantallas de login/registro o rutas protegidas mediante los botones del navegador.
- Protección y manejo de rutas mediante `PublicRoute` y `ProtectedRoute`, además de Contextos dedicados (`AuthContext`) para mantener la sincronización y la limpieza de datos en toda la sesión.
- Rediseño total de la sección "Mis análisis": Implementación de una interfaz basada en pestañas (Estados Financieros / Catálogos) con un flujo de trabajo sin modales (inline forms). El formulario de creación reemplaza completamente el contenido principal para mejorar el enfoque del usuario. Incluye gestión de estados locales para análisis y catálogos, con selectores embebidos y badges de visibilidad (Público/Privado).
